# Telegram Jung2 Bot - Go Migration Plan

## Mission

Complete `telegram-jung2-bot` as a Go-only service while preserving the
production contract that existing chats, queues, schedules, and data tables
depend on.

During migration, reference fixtures and source snapshots may be used to prove
behaviour. They are not part of the desired finished project. The finished
project should read as a standalone Go service: no migration-era wording,
reference-runtime files, migration-only fixtures, or transitional scaffolding
should remain in user-facing docs, package names, exported APIs, or runtime
paths.

Final acceptance requires:

- Full production feature parity for supported bot behaviour.
- `make test` passing with the required 100% Go statement coverage.
- `make lint` passing.
- Production adapters and worker handlers wired in the default runtime path.
- A cleanup pass that removes migration-only artifacts and wording.

## Production Contract

- Preserve DynamoDB table names, key names, attribute names, and value formats.
- Preserve `MESSAGE_TABLE` keys: partition key `chatId`, sort key
  `dateCreated`.
- Preserve message TTL behaviour: messages expire after 7 days.
- Preserve `CHATID_TABLE` key `chatId` and attributes `chatTitle`,
  `enableAllJung`, `offTime`, and `workday`.
- Preserve SQS action names and payload shape.
- Preserve both SQS message attribute casings:
  `messageAttributes.action.stringValue` and
  `messageAttributes.action.StringValue`.
- When both SQS casings are present, `stringValue` wins over `StringValue`.
- Preserve Lambda SQS event wrapper handling:
  `Records[0].messageAttributes`.
- Preserve Telegram command names, aliases, response text, ordering, and
  truncation behaviour.
- Preserve independent command matching: one Telegram message may enqueue
  multiple supported commands in the fixed command-check order.
- Preserve the 3800-character Telegram response safety limit with valid UTF-8,
  body ellipsis behaviour, and footer preservation.
- Preserve workday bitmask values:
  `Sun=1`, `Mon=2`, `Tue=4`, `Wed=8`, `Thu=16`, `Fri=32`, `Sat=64`.
- Preserve existing `dateCreated` parsing for the stored UTC+8 offset format
  before normalising time internally.
- DynamoDB reads must handle pagination until `LastEvaluatedKey` is empty.
- Logs must avoid unnecessary Telegram message text.

## Current Layout

The Go module lives at the repository root. The service executable lives under
`cmd/telegram-jung2-bot`, and private Go packages live under `internal/`. Do not
create a `go/` subdirectory; Buck2 target visibility controls package access.

Migration-only reference material is allowed until production adapter parity is
complete. It must be removed before the final cleanup gate.

## Team Rules

- Prefer independent modules first; defer app wiring and production adapters
  until package contracts are stable.
- Keep domain packages free of AWS SDK clients, Telegram HTTP clients, HTTP
  server code, environment readers, and app wiring.
- Use package-local interfaces, fakes, or generated mocks.
- Do not add shared `testutil`, shared mocks, or abstraction packages until
  repeated code proves they are needed.
- Every behaviour change should identify the production contract or reference
  fixture it preserves.
- Buck2 is the long-term build and test system; keep Buck targets deterministic.
- Use Buck2's official `prelude//go/tools/gobuckify:gobuckify` for vendor BUCK
  generation.

## Module Contracts

### `config`

- Owns required environment variables, default values, table names, queue URLs,
  Telegram token, AWS region, logging level, local endpoint overrides, server
  addresses, timeouts, and numeric limits.
- Public API: `Load(env map[string]string) (Config, error)`.

### `workday`

- Owns fixed workday bitmask values, parsing stored workday values, and checking
  whether a date is enabled for work.
- Public API: `Parse(mask int) (Workdays, error)`.

### `telegram`

- Owns Telegram update, chat, user, message, administrator models, HTTP client,
  and report truncation.
- Public API: `ParseUpdate`, `Client.SendMessage`, and
  `Client.GetChatAdministrators`.

### `queue`

- Owns SQS action model, stable action names, long polling, enqueueing,
  decoding, dispatch, and deletion.
- Public API: `Action`, `Consumer.Poll`, `Producer.Enqueue`,
  `DecodeMessage`, and `DecodeEvent`.

### `command`

- Owns supported Telegram commands, validation, argument parsing, and conversion
  to queue actions.
- Public API: `Parse`, `ParseAll`, and `ActionFor`.

### `dynamodb`

- Owns request shapes, pagination helpers, common error classification, and
  sanitised logging fields.

### `message`

- Owns stored message model, TTL calculation, timestamp formatting/parsing,
  message repository, and statistics query windows.

### `chat`

- Owns chat settings model, repository, defaults, malformed row handling, and
  due-chat listing. `enableAllJung` must not filter scheduled off-work
  eligibility.

### `statistics`

- Owns ranking calculations, report models, report rendering, ordering rules,
  Moment-like relative time wording, and truncation.

### `schedule`

- Owns off-work scheduling, schedule identity, due report handling, and skip
  reasons.

### `httpserver`

- Owns HTTP routing, webhook endpoint, health checks, request validation,
  response status codes, stage-route responses, and dependency-oriented
  handlers.

### `worker`

- Owns queue action dispatch, handler contracts, delete-on-success behaviour,
  and retry-on-failure preservation.

### `app`

- Owns application wiring, dependency construction, worker lifecycle, startup,
  and graceful shutdown.

## Quality Gates

A package is complete when:

- Its public API is narrow and documented by tests.
- Relevant production-contract tests have coverage.
- Go statement coverage remains at 100% through `make test` or
  `make test-coverage`.
- It has no direct dependency on unrelated adapters or app wiring.
- It handles contexts for network, AWS, Telegram, and shutdown-aware operations.
- Errors are wrapped with useful context using `%w`.
- Logs use `log/slog` for structured output where logging is needed.
- Package-local fakes/mocks are used instead of live external services.
- Buck targets remain deterministic and updated for new Go files.

A vertical slice is complete when:

- The composed Go flow matches the production contract for the relevant fixture.
- `make test` passes before production replacement or cleanup.
